- Risk-Aware V1 is efficiency-support evidence, not the primary safety theorem.
- HCE is a configuration-specific recovery-efficiency improvement, not a new recovery-capability claim.
- SAFC/R1 is paused; its shadow/interface artifacts are retained as diagnostic provenance only.
- The original SAFER-Splat baseline is a comparator, not an innovation claim in this freeze.
- AdaptiveV1, warning slowdown, VANS, V4-B, and GTEP are diagnostic or negative evidence unless a later, separately registered evaluation promotes them.
- Trial-20 is a local recovery-exhaustion boundary under its tested contract. It is not a planner result, a safe-halt result, or a global impossibility proof.

## 4. Cross-dataset staged protocol

1. **G0 portability audit:** establish dataset/checkpoint/start-goal/data-path reachability before any method claim.
2. **G1 baseline reproduction:** run the unmodified comparator under frozen hardware/logging and record all trial outcomes.
3. **G2 failure taxonomy:** classify observed outcomes using the frozen F1-F8 taxonomy before selecting an optional module.
4. **G3 selective module evaluation:** evaluate only modules justified by G2, with preregistered fixed parameters and separate original/post-repair reporting.

No G1--G3 result may change a configuration after outcomes are observed. A conflict in H, N, margin, seed, timestep, dynamics, radius, collision predicate, or trial order blocks the corresponding formal comparison.

## 5. Same-dataset invariants

The following must match within a formal comparison: checkpoint, start/goal source, trial ordering, seed (when the source exposes it), timestep, dynamics, robot radius, collision predicate, evaluation `h`, warm-up treatment, hardware description, and log schema. Missing source values remain `unresolved`; they are never guessed.

Original benchmark results, post-repair results, development cohorts, held-out cohorts, and targeted-diagnostic cohorts are distinct registries and cannot be pooled.

## 6. Trial-20 exclusion rule

Trial 20 is retained in its original activation order only for diagnosis. It is excluded from candidate tuning and does not become a development tuning case. Any future active redesign requires a new protocol, separate controls, and fresh held-out selection.

## 7. Randomness rule

Only source-declared random seeds may be recorded as fixed. If no source record exists, the field is `unresolved`; a later experiment must preregister one before it is formal. No historical seed is reconstructed from outcomes.
